BUDGET AND FINANCE APP WORKFLOW
===============================

Goal
----
The user tracks personal wealth, income, and spending during the year.
The app does not force predefined expense categories. The user creates
their own budget categories and assigns calculated expenses to them.

Previous Total Assets + Total Income During Period - Current Total Assets

Notes:
- Transfers between the user's own accounts should not be counted as income or expense.
- If the user buys an asset that is tracked by the app, it should increase assets instead of being treated only as spending.
- If calculated expenses are negative, the app should ask the user to review balances, income, or missing assets.

MAIN USER FLOW
--------------
1. User creates budget categories and yearly budget amounts.
2. User adds assets, cash, bank accounts, and other accounts to track.
3. User periodically updates balances for all tracked assets.
4. User enters income received during the period.
5. App calculates expenses from asset difference and income.
6. App asks user to assign calculated expenses to user-created categories.
7. App updates yearly budget usage.
8. App shows used budget, remaining budget, unassigned expenses, and asset changes.
